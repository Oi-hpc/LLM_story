我想制作一个H5的基于LLM的交互式叙述游戏，故事走向由 AI 决定，以生化危机的剧情作为梗概，让 AI 向玩家描述剧情，并在一些节点询问用户该怎么做，并根据用户输入推进剧情。以下是详细的设计点。

# 玩法设计
1. AI 除了给出指定选项外，还可以让用户进行自定义输入行动方案；
2. 为了保持良好的游玩节奏，采用章节模式设计；同时可以避免上下文过长导致 AI 失忆，每次一个章节完成后，让 AI 把上一章节的重要抉择和角色状态进行简要总结后，打包传给 AI 进行新一轮的章节游戏，每一章节的体量控制在 10 分钟左右；
3. 提供回溯功能，每一轮对话时都把到该次对话为止的上下文缓存下来，当用户需要回溯到某次选择时，调用到该次选择的上下文即可，不可跨章节回溯；
4. 要有优质的提示词，保证用户沉浸式体验，自由度高但也不会剧情乱飞；
5. 可以保存，用户退出后重新进入可以继续上一次游戏。

# 用户体验与交互
为了增强用户体验，必须要有良好的前端页面：
1. 要有游戏地图；根据每一个章节预先离线生成地图，由 AI 判断当时用户状态来实时标注用户在地图中的位置以及用户可以看到的地图范围；
2. 要有音乐；由舒缓的、紧张的、悬疑的、优美的等不同风格的音乐，让 AI 根据当前剧情状况来选择 BGM；
3. 要有数值系统的前端展示，该数值系统的展示界面要符合生化危机的背景故事，制造沉浸感；
4. 整体的画风、字体和页面风格都要符合背景故事。

# 数值系统与剧情规则
- **数值项**：明确定义数值系统包含的项，例如：生命/体力、弹药、关键道具数量、与关键 NPC 的关系值等；是否支持「死亡/失败」分支（如生命归零触发 Game Over 或分支剧情）需在设计中约定。
- **与 LLM 的约定**：数值由 AI 在每轮回复中通过结构化字段给出（如 JSON 中的 `state_update`），前端解析后更新 UI，避免纯自然语言解析不稳定。
- **剧情边界**：提示词中约定「不可脱离生化危机世界观」「重大分支（如角色生死）需在章节内收束或写入摘要」，避免剧情乱飞。

# 技术架构与数据流
- **前端**：H5 技术栈（如 Vue3/React + 响应式布局），适配移动端与桌面；若偏轻量可考虑纯 HTML/CSS/JS。
- **LLM 调用**：通过后端代理调用大模型 API（如 OpenAI/Claude/国产 API），避免暴露 Key；或使用 Serverless/云函数。
- **数据与存储**：
  - **存档**：当前进度（章节号、章节内回合号、章节摘要、角色状态、背包等）存 localStorage 或可选后端账号体系。
  - **回溯**：每回合完整上下文（或状态快照）在内存 + 当前章内持久化，回溯时按「回合点」恢复，不跨章。
- **地图与资源**：章节地图为预置静态资源（图片/SVG）；BGM、音效为预置音频文件，按「情绪标签」由前端切换。

# LLM 接口与数据格式
- **请求内容**：系统提示词（人设、世界观、输出格式）+ 当前章节摘要（若非第一章）+ 本章节历史对话与状态 + 用户本轮输入（选项或自定义）。
- **响应格式**（建议结构化，便于前端与地图/BGM 联动）：
  - 叙述正文（用于主剧情框）；
  - 可选选项列表（如 3 个固定选项 + 是否允许自定义输入）；
  - 状态更新（生命、弹药、道具等）；
  - 地图信息：当前节点/区域 ID、可见范围 ID 或坐标（与预置地图配置对应）；
  - BGM 标签：如 `tense` / `calm` / `mystery` / `action`，前端据此切歌。
- **章节结束**：由 AI 明确返回「章节结束」标记 + 本章摘要（重要抉择、角色状态、关键道具），供下一章初始化使用。

# 地图与 BGM 实现约定
- **地图**：每章一张预生成地图资源；前端根据 AI 返回的位置标识（如 `area_id`）高亮当前所在区域，并根据「可见范围」做遮罩或高亮显示。地图与区域 ID 需在章节设计阶段与提示词/剧情节点对齐，避免 AI 返回无法映射的坐标。
- **BGM**：预定义若干首按情绪分类的 BGM；AI 在响应中只返回情绪标签，前端维护「标签 → 曲目」映射，实现淡入淡出切换。

# 回溯与存档细则
- **回溯**：仅限当前章节；回溯到某一「回合」时，恢复该回合的完整状态（对话、选项、数值），并丢弃该回合之后的上下文与缓存。回溯后的再次推进，需重新调用 LLM（基于回溯点的上下文继续），而不是简单回放历史。
- **存档**：需保存章节索引、章节内回合索引、章节摘要、当前数值与背包、当前章的回溯点列表（或上下文快照列表）。若仅本地存储，注明「不支持跨设备」；若后续做账号，可补充「云端存档」为扩展需求。

# 异常与边界处理
- **网络与 API**：请求超时、鉴权失败、限流时，前端提示「叙事暂时无法继续」，支持重试；可考虑关键步骤的「离线兜底文案」（如固定选项）作为可选。
- **内容与安全**：若用户输入或模型输出触发审核，需有统一提示（如「请换一种说法再试」）并禁止将违规内容写入存档或摘要。
- **性能与体验**：长叙述可采用流式输出（SSE）逐步展示；章节切换时预加载下一章地图与 BGM，减少卡顿。

# 提示词与内容规范
- **系统提示词要点**：身份（叙事者/游戏主持人）、世界观（生化危机梗概）、输出格式（必须包含叙述 + 选项 + 状态 + 地图标签 + BGM 标签 + 章节结束标记）。
- **禁止项**：脱离世界观、无结构化的数值描述、跨章节未收束的重大分支。
- **章节摘要的必含项**：关键抉择、角色生死与状态、关键道具、下一章入口情境。

# 可选补充（按需加入）
- **多结局**：是否根据关键抉择在终章走向不同结局；若需要，可在「数值系统与剧情规则」中增加「关键抉择与结局分支表」。
- **无障碍**：字体与对比度需符合生化危机风格的同时，可注明「支持系统字体缩放」以兼顾可访问性。
- **国际化**：若未来有多语言，可注明「叙述与选项文案、提示词需支持多语言版本」。
